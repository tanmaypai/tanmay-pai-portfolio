Last Man Standing

Abstract

The inspiration for this project was to encourage a sedentary person to break up extended bouts of sitting with short bouts of standing (without nagging). Last Man Standing is a modern day scavenger hunt game wherein 2 players alternate turns between taking a picture of a particular object. The twist here is that your opponent decides what object you need to find. The challenge is to provide a commonplace object not found easily (e.g. fire alarm switch). Scavenging for the object forces you to move away from your desk but it just as easily allows you to come back. Other features include playing socially (through Facebook), reward points and a flagging system to combat cheating.


What I Did

– Conceived the novel idea for the app which attempts to solve a real world problem
– Sketched out low-fidelity prototypes and tested them with a sample of the end users
– Created storyboards and incorporated different scenarios into the design
– Designed a layout of UI screens that would be consistent across different mobile screen sizes
– Increased the social quotient of the app by integrating several Facebook features like inviting friends and sharing photos
– Wrote a module for flagging/unflagging items to prevent cheating

  • Course: Mobile Application Design and Development
  • Duration: September 2012 - December 2012

Mosnuic

Abstract

The ultimate purpose of Mosnuic is to create a photographic mosaic. Wikipedia defines a photographic mosaic as a photo that has been divided into equal sized rectangular sections (known as tiles), each of which is replaced with another photograph that matches the target photo. When viewed at low magnifications, the individual pixels would appear as the primary image. When viewed at high magnifications, it would be evident that the image is in fact made up of a large number of smaller images.


What I Did

– Analysed requirements set forward for the project by the client (Professor Will Clinger)
– Developed a module for parsing and robust validation of the input arguments presented through the command line
– Developed a randomized algorithm to best fit each tile in the original image and render the final mosaic
– Presented a code review of the application highlighting important modules and algorithms

  • Course: Software Development
  • Duration: September 2012 - December 2012

Mosnuic

Abstract

This project aims to automate the path finding process between two locations. Constraints, such as obstacles are introduced to achieve a realistic path. Performance comparison is made between the various Algorithms. Different Heuristics which are used to guide them to goal node are also presented and compared. The process of Machine Learning is implemented which learns the path and stores in the database when next time if same path needs to be found it is loaded directly from the database and computational overhead of calculating it through algorithm is reduced. Implementation of A*, D*(which checks for obstacles in real time) is practically shown with the help of a small game.


What I Did

– Conceived the design for the GUI in Visual Basic
– Implemented the A* and D* algorithms using a heuristic approach
– Conceived the idea of saving the path in a database for ease of use and reducing computational overheads
– Delivered the final presentation and defended the abstract in front of the college panel

  • Course: Final Year Project at University of Mumbai
  • Duration: August 2009 - May 2010
